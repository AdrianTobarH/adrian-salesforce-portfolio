Context

Business Problem

  • API credentials hardcoded in Apex — critical security vulnerability
  • No reusable integration pattern — each system required bespoke code
  • External data sync running on nightly batch only — no real-time option
  • Zero error handling — failed callouts silent with no retry mechanism
Architecture Strategy

Solution Design

  • Named Credentials + Auth Provider for zero-secret Apex callouts
  • Apex HttpCallout service class — single integration point for all APIs
  • Platform Events for async real-time sync decoupled from transactions
  • Custom object logging layer for callout auditing and retry management
Integration Layers

Technical Implementation

  • OAuth 2.0 Auth Provider managing token refresh automatically
  • Named Credential as abstraction layer — no URL or credential in code
  • Apex Queueable chain for retry-safe async callout architecture
  • External Services for no-code API schema registration where viable
Security Model

Credential Architecture

  • All credentials stored in Named Credentials — zero exposure in Apex
  • Per-user credential delegation via User-level Named Credentials
  • CSP Trusted Sites and CORS configured for outbound security
  • Callout log with field-level encryption on sensitive response data
Scalability

Reusable Architecture

  • Single HttpCallout service class supports any REST endpoint via config
  • Custom metadata for endpoint registration — new APIs without code
  • Platform Event bus absorbs volume spikes without governor limit risk
  • Retry queue with exponential backoff for transient failure resilience
Business Impact

Delivered Outcomes

  • 100% of API secrets removed from codebase — security audit passed
  • Reusable layer reduced new integration time from weeks to days
  • Real-time sync replaced nightly batch — latency cut from 24h to seconds
  • Callout logging surfaces integration health in executive dashboard
